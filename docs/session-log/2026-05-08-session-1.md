## 2. Eight findings documented in ADR-014

All findings are captured in `docs/architecture/adr/ADR-014-live-deployment-corrections.md`:

| # | Finding | Prior assumption | Live reality |
|---|---|---|---|
| 1 | OmniRoute is remote | Local systemd unit on VPS | Remote host `omniroute.infinitycore.space:8443` |
| 2 | Hermes config format | `agent.model` / `agent.fallback_models` | `model.default` / `model.provider: custom` / `model.api_key` / `model.base_url` |
| 3 | OmniRoute auth key | Separate `OMNIROUTE_API_KEY` | `FIREWORKS_API_KEY` is the auth key |
| 4 | Model IDs | `accounts/fireworks/models/deepseek-v4-pro` format | Short-form `deepseek-v3p2` works on deployed OmniRoute |
| 5 | devassist needs HOME | No home dir specified | Hermes crashes with `PermissionError` without writable HOME |
| 6 | StartLimitIntervalSec | In `[Service]` section | Belongs in `[Unit]` section |
| 7 | TELEGRAM_ALLOWED_USERS | Placeholder value | Must be real numeric Telegram user ID |
| 8 | Config templates | Copied as-is | Must be rendered via `render_runtime_configs()` |

### 9.3 PR creation workaround

`git_pr(action="create")` returns "Resource not accessible by personal access token" — known issue documented in PR #143/144/145/146 commit messages. Workaround:

```
# 1. Compose payload JSON
cat > /tmp/pr-payload.json <<'EOF'
{
  "title": "...",
  "head": "<branch>",
  "base": "main",
  "body": "..."
}
EOF

# 2. Create via REST API
curl -sS -X POST \
  -H "Authorization: token $GITHUB_TOKEN_DEVELOPER_ASSISTANT" \
  -H "Accept: application/vnd.github+json" \
  -H "X-GitHub-Api-Version: 2022-11-28" \
  https://api.github.com/repos/OpenClown-bot/developer-assistant/pulls \
```

# TKT-002: Add GitHub Actions CI Baseline

## 1. Scope

Add a minimal GitHub Actions workflow that runs repository docs validation on pull requests and pushes to the default branch.

## 2. Non-scope

- Do not add language-specific build/test jobs before production code exists.
- Do not configure Hermes, Telegram, or deployment secrets.
- Do not configure `pr-agent` in this ticket.
- Do not implement merge automation.

## 3. Required Context

- `AGENTS.md`
- `CONTRIBUTING.md`
- `docs/architecture/ARCH-001.md`
- `docs/tickets/TKT-001.md`
- `scripts/validate_docs.py`

## 4. Acceptance Criteria

- [ ] A GitHub Actions workflow exists under `.github/workflows/`.
- [ ] The workflow runs `python scripts/validate_docs.py`.
- [ ] The workflow triggers on pull requests and pushes to the default branch.
- [ ] The workflow uses a maintained Python setup action or available runner Python.
- [ ] The workflow does not require repository secrets.
- [ ] Local validation command still passes.

## 5. Allowed Files

- `.github/workflows/`
- `docs/tickets/TKT-002.md` Execution Log only

## 6. Test/Validation Requirements

- Run `python scripts/validate_docs.py` locally.
- If feasible, use GitHub Actions syntax validation available in the environment; otherwise note that CI execution must be verified after pushing.

## 7. PR Requirements

- Link this ticket.
- Include local validation output.
- Mention that no secrets are required.

## 8. Risks

- Workflow may fail on GitHub if path assumptions differ from local environment.
- Missing default-branch name could require adjustment after the GitHub repository is created.

## 9. Dependencies

- TKT-001 should be complete or already provide a working validator.

# TKT-004: Create PR and Review Artifact Templates

## 1. Scope

Create templates for implementation PR descriptions and Reviewer LLM review artifacts so every implementation PR can be traced to a ticket, CI result, acceptance criteria, risks, and reviewer verdict.

## 2. Non-scope

- Do not configure automated PR review tools.
- Do not create production review automation.
- Do not modify GitHub repository settings.
- Do not implement merge automation.

## 3. Required Context

- `AGENTS.md`
- `CONTRIBUTING.md`
- `docs/architecture/ARCH-001.md`
- Existing `docs/reviews/` content, if any

## 4. Acceptance Criteria

- [ ] A PR description template exists and includes linked ticket, summary, acceptance criteria status, tests run, known limitations, and risk notes.
- [ ] A Reviewer artifact template exists and includes PR reference, ticket, architecture/ADR references, CI status, findings, verdict, and residual risks.
- [ ] Reviewer verdict values match `pass`, `pass_with_changes`, and `fail`.
- [ ] Templates include fields for Telegram/founder approval when required by ARCH-001.
- [ ] Templates do not require secrets or private data.
- [ ] `python scripts/validate_docs.py` passes.

## 5. Allowed Files

- `.github/pull_request_template.md`
- `docs/reviews/`
- `docs/tickets/TKT-004.md` Execution Log only

## 6. Test/Validation Requirements

- Run `python scripts/validate_docs.py`.
- Manually verify templates include all required PR and review-gate fields from `CONTRIBUTING.md` and ARCH-001.

## 7. PR Requirements

- Link this ticket.
- Include local validation output.
- Explain where reviewers should write final artifacts.

## 8. Risks

- Templates may become too verbose and reduce compliance.
- Missing fields could weaken traceability.

## 9. Dependencies

- None.

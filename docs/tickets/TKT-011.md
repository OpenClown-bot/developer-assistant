# TKT-011: Run First Telegram-to-PR Orchestration Trial

## 1. Scope

Run the first end-to-end trial of the revised v0.1 workflow: founder interaction through Telegram/Hermes, ticket selection, branch/PR creation, CI/docs validation, Reviewer artifact, progress report, and founder merge acknowledgement.

## 2. Non-scope

- Do not perform live VPS deployment.
- Do not bypass review gates.
- Do not merge without explicit founder acknowledgement.
- Do not introduce OpenClaw.

## 3. Required Context

- `AGENTS.md`
- `CONTRIBUTING.md`
- `docs/orchestration/SESSION-STATE.md`
- `docs/architecture/ARCH-001.md`
- Relevant ADRs in `docs/architecture/adr/`
- The selected ready ticket

## 4. Acceptance Criteria

- [ ] A single ready ticket is selected for the trial.
- [ ] The trial begins from a Telegram founder command or message handled through Hermes.
- [ ] Work is performed in a branch and PR, not direct main changes.
- [ ] PR description follows the repository PR template.
- [ ] CI or local docs validation result is captured.
- [ ] Reviewer artifact is written under `docs/reviews/` with an allowed verdict.
- [ ] A Telegram progress report summarizes the result and any blocker.
- [ ] Founder approval is requested before merge.

## 5. Allowed Files

- Files allowed by the selected ready ticket
- `docs/reviews/`
- `docs/orchestration/SESSION-STATE.md`
- `docs/tickets/TKT-011.md` Execution Log only

## 6. Test/Validation Requirements

- Run `python scripts/validate_docs.py`.
- Record PR URL and validation status in the review artifact or orchestration state.
- Document sanitized Telegram smoke test evidence.

## 7. PR Requirements

- Link this ticket and the selected trial ticket if separate.
- Include tests run.
- Include known limitations and risks.

## 8. Risks

- GitHub remote or PAT may be unavailable.
- Telegram/Hermes smoke test may require manual environment setup.
- Trial may reveal missing validation rules or prompt ambiguity.

## 9. Dependencies

- TKT-001 through TKT-009 should be complete or explicitly waived by the user for a narrower trial.
- GitHub remote and permissions must be available.
- Telegram/Hermes runtime must be available.

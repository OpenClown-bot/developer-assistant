# TKT-012: Source-Review Credential-Bearing Hermes Capabilities

## 1. Scope

Perform and document the minimum source review required before credential-bearing Hermes Telegram and GitHub capabilities may be used for production runtime work.

## 2. Non-scope

- Do not implement Telegram command handling.
- Do not implement GitHub repository or PR automation.
- Do not add, print, or persist real credentials.
- Do not perform a full security audit of all Hermes Agent code.
- Do not enable deferred marketplace, community, or project-local plugins.

## 3. Required Context

- `AGENTS.md`
- `CONTRIBUTING.md`
- `docs/architecture/ARCH-001.md`
- `docs/architecture/HERMES-RUNTIME-CONTRACT.md`
- `docs/architecture/HERMES-SKILL-ALLOWLIST.md`
- `docs/architecture/adr/ADR-001-platform-foundation.md`
- `docs/architecture/adr/ADR-003-plugin-supply-chain.md`
- `docs/reviews/RV-CODE-009.md`
- Hermes Agent source and documentation for the pinned v0.1 version or commit

## 4. Acceptance Criteria

- [ ] The review covers the Telegram gateway capability used by TKT-006 and records whether it may receive `TELEGRAM_BOT_TOKEN` in production.
- [ ] The review covers the GitHub PR workflow capability used by TKT-008 and records whether it may receive `GITHUB_TOKEN` or `GH_TOKEN` in production.
- [ ] The review checks for obvious credential exfiltration vectors, unsafe credential logging, broad network/file access assumptions, and unauthorized repository access paths.
- [ ] `docs/architecture/HERMES-SKILL-ALLOWLIST.md` is updated with source review result, reviewer/date, and any remaining constraints for reviewed capabilities.
- [ ] If a capability does not pass source review, the ticket documents the blocker and a safe fallback or follow-up path.
- [ ] The review does not introduce or expose real secret values.
- [ ] `python scripts/validate_docs.py` passes.

## 5. Allowed Files

- `docs/architecture/HERMES-SKILL-ALLOWLIST.md`
- `docs/tickets/TKT-012.md` Execution Log only

## 6. Test/Validation Requirements

- Run `python scripts/validate_docs.py`.
- Manually verify that no secret values, chat IDs, PATs, API keys, or private repository credentials are written to repository artifacts.
- If a source-inspection command is used, document the command without printing secrets.

## 7. PR Requirements

- Link this ticket.
- Include validation output.
- Summarize reviewed files/modules or documentation paths.
- State which capabilities remain blocked or are cleared for production credential-bearing use.
- Include risk notes for any incomplete review area.

## 8. Risks

- Hermes Agent is a large runtime, so this review can only clear the minimum credential-bearing paths needed for v0.1.
- Source layout or bundled skill names may differ from the allowlist assumptions and require ticket updates.
- Marking a capability as reviewed too broadly could create false confidence.
- Overly strict review results may delay Telegram or GitHub implementation until a safer fallback is defined.

## 9. Dependencies

- TKT-009 must be done.
- TKT-006 and TKT-008 must remain draft for production credential-bearing runtime use until this ticket is done or the user explicitly approves a narrower non-production/dry-run path.

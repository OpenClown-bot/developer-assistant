### 1.0 Original scope (preserved as historical record, v0.1.0 verbatim)

Perform the first live deployment of `developer-assistant` onto the Founder-owned Ubuntu 22.04 LTS VPS using the TKT-020 install scripts, fix any issues encountered, inject credentials through the approved path, and verify that the system reaches a healthy operational state ready for the TKT-011 live trial.

This is the bridge ticket between "scripts in repo, never tested on a real VPS" and "system is live and TKT-011 can run".

### 1.1 Deployment steps (Executor performs on VPS via opencode)

1. **Clone repo** on VPS: `git clone https://github.com/OpenClown-bot/developer-assistant.git /srv/devassist/repo` using `PROJECT_GITHUB_PAT` in the remote URL for authentication.
2. **Run install script**: `sudo bash /srv/devassist/repo/scripts/install-self.sh`. Expect: the script creates the filesystem layout, installs Hermes, renders systemd units, but does NOT start any runtime (per SELF-DEPLOYMENT-CONTRACT §3 `install` gate).
3. **Fix any issues** encountered during install. The TKT-020 scripts were written offline and have never run on a real VPS. Likely breakages: missing system packages, incorrect paths, permission issues, template rendering errors, Python dependency mismatches, Hermes version incompatibilities. Each fix is committed to the branch.
4. **Inject credentials** into `/srv/devassist/secrets/SELF-DEPLOY.env` (mode 0600, owner devassist:devassist). The env file must contain at minimum:
   - `TELEGRAM_BOT_TOKEN=<value>` — read from `TELEGRAM_BOT_TOKEN` env var in the Executor's opencode session (NOT hardcoded in the NUDGE or committed)
   - `PROJECT_GITHUB_PAT=<value>` — read from `PROJECT_GITHUB_PAT` env var in the Executor's opencode session
   - `FIREWORKS_API_KEY=<value>` — read from `FIREWORKS_API_KEY` env var (or equivalent OmniRoute provider key)
   - Any other keys required by OmniRoute / Hermes as per SELF-DEPLOYMENT-CONTRACT §4 and ADR-011.
   The Executor reads these from its environment and writes them to the file. The values MUST NOT appear in git commits, PR bodies, NUDGE text, or any repository artifact.
5. **Verify**: `sudo bash /srv/devassist/repo/scripts/verify-self.sh`. Expect: all health invariants pass, `operational.db` is writable, all five health endpoints respond on `127.0.0.1:8181..8185/health`.
6. **Start runtimes**: `sudo systemctl start devassist.target`. Per SELF-DEPLOYMENT-CONTRACT §3, this requires explicit Founder approval. The Executor MUST NOT start runtimes autonomously — it prepares everything and requests Founder to run the start command.
7. **Post-start verify**: After Founder starts the target, run `dev-assist-cli status --format human` and confirm all five runtimes report `running` or `degraded` (not `down` / `unknown`).

### 1.2 Credential injection contract

- The Executor opencode session on the VPS receives credentials through **environment variables**, not through NUDGE text or prompt content.
- The NUDGE tells the Executor: "Read `TELEGRAM_BOT_TOKEN`, `PROJECT_GITHUB_PAT`, and `FIREWORKS_API_KEY` from your environment and write them to `/srv/devassist/secrets/SELF-DEPLOY.env` with correct ownership and permissions."
- The Executor MUST NOT print, log, or commit any credential value. If a value must be confirmed present, the Executor reports "key TELEGRAM_BOT_TOKEN is set (length N)" or similar, never the value itself.
- The Founder sets these env vars in the opencode session before the Executor starts.

### 1.3 Out of scope (Founder performs)

- Installing opencode CLI on the VPS
- Configuring OmniRoute (`/opt/omniroute/`, provider registry, API keys in omniroute.db)
- Installing system-level prerequisites not covered by the install script (e.g., Python 3.11+, pip, venv) — if the install script is missing these, the fix is IN scope (Executor updates the script)

## 8. Risks

- The install script may have many issues on a real VPS — this ticket could require significant debugging. Mitigated: the Executor runs on the VPS via opencode and can iterate interactively.
- Hermes Agent version may not match assumptions in the scripts. Mitigated: the Executor can pin the version and adjust.
- The VPS may lack system packages (Python, sqlite3, etc.) that the install script assumes. Mitigated: the Executor adds `apt-get install` calls to the script.
- Credential injection is sensitive — any leak in git history is irreversible. Mitigated: the Executor writes values from env, never commits them; NUDGE text contains key names only.
- The `devassist` system user created by the script may have incorrect permissions. Mitigated: verify step checks ownership and mode.
- OmniRoute must be configured by the Founder before runtimes can make LLM calls. If OmniRoute is not ready, runtimes will start but report `degraded` on health checks. This is acceptable — TKT-011 can still run if at least Telegram lane passes.

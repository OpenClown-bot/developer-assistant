## 5. Findings

### HIGH — Write-zone violation: `docs/reviews/RV-022.md`

- **Location**: `docs/reviews/RV-022.md`
- **Description**: This file is a review artifact for PR #100 (TKT-022 work-queue schema). It is **not** listed in TKT-021 §5 Allowed Files. CONTRIBUTING.md § Roles requires strict write-zone compliance per ticket scope.
- **Recommendation**: `git rm docs/reviews/RV-022.md` and commit the removal before merge. This finding blocks merge-safe.

### MODERATE — `_ROLE_SKILLS` duplicated across `runtime_layout.py` and `runtime_check.py`

- **Location**: `src/developer_assistant/runtime_layout.py:75-119` and `src/developer_assistant/runtime_check.py:23-67`
- **Description**: The same constant table appears in both modules with identical content. If one is updated without the other, skills verification will silently diverge, breaking runtime checks against rendered configs.
- **Recommendation**: Extract `_ROLE_SKILLS` to a single source (e.g., `runtime_layout.py` exports it and `runtime_check.py` imports it) or create a shared `constants.py` module. This does **not** violate any AC (AC #16 only requires the table "in code"), but it is a maintenance risk.

### LOW — `etc/runtime-templates/SOUL.md.tmpl` is dead data

- **Location**: `etc/runtime-templates/SOUL.md.tmpl`
- **Description**: After the Executor refactored `_render_soul_md()` to generate SOUL.md content inline, the shared template file is no longer consumed by any code path. It is listed in TKT-021 §5 Allowed Files so its presence is legal, but it serves no function.
- **Recommendation**: Either wire `_render_soul_md()` to read from the template, or leave a code comment noting it as a placeholder for future use. Not merge-blocking.

## 9. Residual Risks

1. **Drift risk**: `_ROLE_SKILLS` duplicated in two modules. A future ticket that changes skills for one role must remember to update both tables.
2. **Template rot**: `SOUL.md.tmpl` is currently dead data. If a future refactor expects it to exist, the inline `_render_soul_md()` may be overlooked.
3. **Symlink tests skip on Windows**: The `test_all_five_roles_pass_in_fixture_mode` and three operational-db symlink tests skip on Windows (correct platform guard), meaning the full runtime-check integration is only exercised in CI or on Linux. This is acceptable given the VPS deployment target.
